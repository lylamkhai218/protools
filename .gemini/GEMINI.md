# GEMINI.md - Gemini AI Developer Agent Core Configuration

Welcome to the **Gemini AI Developer Agent** production-grade workspace configuration.
This workspace defines specialized agent roles, interactive task commands, coding quality regulations, and validation checklists to ensure premium, maintainable, and highly polished web implementations.

---

## 🛠️ Overview of Workspace

The `.gemini/` structure is decomposed into key areas:
- **`commands/`**: Operational prompt accelerators (`/spec`, `/plan`, `/build`, `/test`, `/review`, `/deploy`, etc.) to streamline development phases.
- **`agents/`**: Core personas for vertical slicing, specialized review, and architecture.
- **`rules/`**: Mandated strict rulesets covering clean code, naming styles, error isolation, security safeguards, and UI styling.
- **`skills/`**: Expert reference implementations for complex actions (e.g., TDD, structural migrations, API integrations).
- **`references/`**: Pre-merge QA checklists (security, accessibility, performance).

---

## 🧭 Interactive Development Workflow

Our agentic cycle consists of six integrated steps to eliminate regression and deliver absolute architectural integrity:

1. **Define (`/spec`)**
   - Translate instructions into a solid technical specification centering on objective boundaries.
2. **Plan (`/plan`)**
   - Decompose specifications into vertical database, backend, and frontend slices.
3. **Build (`/build`)**
   - Code incrementally using pristine typography, flexible page components, and robust types.
4. **Verify (`/test`)**
   - Execute linter, type-checker (`tsc`), and test runners, solving compilation anomalies immediately.
5. **Review (`/review`)**
   - Perform a comprehensive multi-axis review focusing on performance, accessibility, and types.
6. **Ship (`/deploy`)**
   - Standardize target builds and finalize verification.
