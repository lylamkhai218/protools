# testing.md - Test Standards & Integrity Verification

To maintain pristine engineering standards, implement robust testing patterns.

## 🧪 Testing Guidelines
- **Lint Validation**: Ensure compile passes execute with zero unresolved errors or missing imports.
- **Component Integrity**: Test crucial components (like currency calculators or cart quotes) for valid rendering outputs.
- **No Mock Stubs**: Use real data and types to prevent false-green conditions during active building stages.
